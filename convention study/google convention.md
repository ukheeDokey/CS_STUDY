# Google convention
아직은 이해 안 되는 부분이 많으니 공부하면서 차차 작성해 나가기
-----
- ### 줄바꿈(=개행)
  - 여는 괄호 앞에는 줄바꿈 X
  - 여는 괄호 다음 줄바꿈
  - 닫는 괄호 전 줄바꿈
  - 닫는 괄호 다음 줄바꿈
    - if - else나 ,등으로 이어지는 부분에는 적용 X
    - 메소드, 클래스 등이 끝나는 부분에만 적용
-----
- ### 블럭
  - 빈 블럭 줄바꿈은 해도 되고 안 해도 됨
    - 다만 멀티블럭일 경우 간결한 빈 블럭 형태 {} 사용 X
  - 들여쓰기: +2 스페이스
-----
- ### 배열
  - 배열 초기화는 'bloke-like' 형태로 할 것
  - C 스타일의 배열 선언은 하지 말 것
    - 대괄호는 타입에 붙는다. 변수에 붙으면 안 된다.
      - ex) (String[] args) O, (String args[]) X
-----
- ### 주석
  - //...이나 /*...*/ 스타일 사용
  - /*...*/ 스타일인데 여러 줄일 경우 뒤이어 오는 줄은 다 *로 시작
-----
- ### 네이밍
  - 패키지 이름 : 전부 소문자로 띄어쓰기 하지 않고 붙여서 사용
  - 카멜 케이스 : 중간중간 대문자를 사용하고 나머지는 소문자를 사용, 모양이 낙타의 혹처럼 생겼다고 해서 카멜
    - 클래스 이름 : 대문자로 시작하는 카멜 케이스
    - 함수 이름 : 소문자로 시작하는 카멜 케이스
  - 상수 이름 : 전부 대문자, 각 단어별 띄어쓰기 대신 _(언더바) 활용
