## fork
- fork : 다른 사람의 원격 저장소를 그대로 복제해서 내 저장소를 따로 만드는 방법   
fork 한 저장소는 원본과 연결되어 있음   
fork 한 후 내 저장소를 clone 해서 작업   
보통 프로젝트나 오픈소스에 기여할 때 fork를 많이 사용   
원본 프로젝트와 완전히 분리된 환경에서 작업하여 안전하기 때문
1. 변경사항을 원래 레포지토리에 반영하고 싶으면 원본 저장소에 pull request 보내야 함
2. 보낸 pr은 관리자로부터 승인 받아야 commit, merge 되어 원본에 반영됨   
3. pr 하기 전까진 내 레포에만 적용
---
## clone
- clone : 원격 레포지토리와 나의 로컬 환경을 연결하여 데이터를 복사해 가져오는 기능
- 내가 생성한 원격 저장소를 내 컴퓨터와 연결하여 데이터를 복사
- fork 한 원격 저장소를 내 컴퓨터와 연결하여 데이터를 복사
  이를 통해 원격 저장소에 있는 모든 코드, 히스토리, 브랜치 등을 로컬 환경에 가져올 수 있음
---
### fork 과정 정리
1. 레포지토리에 권한이 없는 사용자가 저장소를 fork
2. fork 한 자신의 저장소에 변경 사항을 적용한 후 push
3. 이후 내 저장소에 있는 브랜치를 원래 저장소에 pull request 요청
4. 승인받으면 원본 저장소에 merge
---
### fork 후 clone 과정
1. 원본 레포지토리를 나의 저장소로 fork
2. 나의 레포지토리를 나의 로컬 pc에 clone
3. 로컬에서 작성 & 수정
4. 원본 레포지토리에 pull request 요청
5. 승인 후 반영
